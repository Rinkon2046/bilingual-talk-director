# 风格规范 · STYLE.md

本规范从参考口播视频逐帧提取,描述"车内口播双语强字幕"风格的精确视觉参数。复刻时严格照此。

## 画布
- 竖版 **1080 × 1920**(9:16)。
- 背景 = 出镜口播视频满屏裁切(`object-fit:cover`)。
- 底部叠一层从透明到 `rgba(0,0,0,.62)` 的渐变压暗,保证字幕可读;顶部轻压暗保证章节条可读。

## 颜色令牌
| 名称 | 值 | 用途 |
|---|---|---|
| 高亮橙 | `#EC6A35` | 关键词高亮、章节进度填充、卡片橙点/pill |
| 主字白 | `#FFFFFF` | 中文字幕、大标题 |
| 译文灰 | `#C7C4BD` | 英文译文 |
| 标签灰 | `#9A958B` | 章节标签、卡片来源标签 |
| 卡片底 | `rgba(22,20,18,0.86)` | 引用/产品卡片背景 |
| 卡片描边 | `rgba(255,255,255,0.10)` | 卡片边框 |
| 轨道灰 | `rgba(255,255,255,0.16)` | 章节进度条底槽 |

> 高亮色**只用一种橙**,且**只给关键词**(每句 0–2 个)。滥用高亮会让画面变脏。

## 字体
- **中文**:`Noto Sans SC`,字幕/标题用 **900**(超粗)。
- **英文 / 标签 / 卡片引文**:等宽 `JetBrains Mono`(或 Roboto Mono),字重 500,带约 `.02em` 字距,常小写。
- 这种"中文超粗黑 + 英文等宽"的对比是该风格的辨识点之一。

## 组件尺寸与排版
### 顶部章节进度条
- 距顶 ~46px,左右内边距 40px,各章等宽横向排列,间距 ~22px。
- 每章 = 标签(等宽 ~21px)+ 下方 4px 高、圆角的进度槽;进度填充为橙色,`transform-origin:left` 横向 scaleX 填充。
- 状态:`done`(已完成,填满、标签偏亮)/ `active`(当前,标签纯白加粗、填充随时间增长)/ 默认(未到,灰、空槽)。

### 底部双语字幕
- 居中,左右内边距 60px,距底 ~150px。
- 第一行中文:900 粗体,字号 ~62px,行高 1.28,带柔和投影。
- 第二行英文:等宽 500,字号 ~30px,灰色,距中文 ~14px。
- 关键词用 `.hl` 橙色(中英都标)。
- 进出:从下方 30px + 透明淡入(.35s, power2.out),结束前 ~0.3s 淡出。

### 暗色卡片(引用 / 产品)
- 左右内边距 ~46px,置于上半部(top ~150px),圆角 ~26px,内边距 ~22px,弱投影 + 轻微毛玻璃。
- 头部一行:橙色圆点 + 等宽来源标签(如 `Naval · Modern Wisdom`、`hedj.io · Personal Career OS`)+ 右上 pill(如 `原片`、`我做的`,橙字 + 橙色半透明底,圆角)。
- 主体三选一/可组合:
  - **图片/截图**:`card-media`,圆角 14px(播客截图、网页截图等)。
  - **引文**:等宽 ~30px,关键词橙色高亮。
  - **产品卡**:内嵌白色"浏览器卡"截图 + 大号品牌字(橙)+ 一行中文 tagline。
- 进出:scale .96→1 + 透明淡入(.45s, power2.out),退场略缩 + 淡出。

### 强调大标题(可选)
- 居中偏上(top ~560px),900 粗体,字号 ~96px,关键词橙色。
- 用在段落之间的"金句/转场"节拍,通常**不与字幕同时出现**。
- 进场:scale .92→1 + 淡入;或逐词出现。

## 动效原则
- 只动**视觉属性**:opacity / y / scale / color。时长 0.3–0.45s。
- 章节进度条匀速 `ease:'none'`;其余用 `power2.out`。
- 任意时刻只突出一个主角(字幕 / 卡片 / 大标题)。
- 节奏跟着语音走:一句话一个 cue,卡片只在被提到时出现 2–4 秒。

## 声音
- 默认用口播视频**原声**作为旁白(背景 `<video>` 不静音)。
- 若要单独配音轨,见 `README_使用说明.md`。
